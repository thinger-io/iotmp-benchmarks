## 2. Embedded Compilation Comparison (ESP32)

### 2.1 ESP-IDF (Primary — Same Framework Comparison)

All programs compiled with **ESP-IDF v5.4** for ESP32 using Docker (`espressif/idf:v5.4`).
Same framework, same toolchain, same target. No TLS. Equivalent functionality:
- Connect to WiFi and authenticate with server/broker
- Define LED actuator (input resource / command handler)
- Define environment sensor (output: temperature + humidity)
- Define uptime resource
- Define reboot action
- Handle communication in main loop

#### Results

| Protocol | Image Size | Lines of Code | Library |
|---|---:|---:|---|
| MQTT | 907,776 B | 122 | esp_mqtt (built-in) + cJSON |
| HTTP | 909,348 B | 109 | esp_http_client (built-in) + cJSON |
| **IOTMP** | **938,844 B** | **48** | iotmp-espidf + iotmp-embedded |
| CoAP | 952,896 B | 128 | libcoap (espressif/coap component) |
| LwM2M | 1,155,124 B | 139 | Anjay (AVSystem/Anjay-esp32-client) |

#### Key Observations

1. **MQTT, HTTP, IOTMP, and CoAP are within 5% of each other.** The ESP-IDF base (WiFi, TCP/IP, TLS, FreeRTOS) dominates at ~880+ KB. Protocol libraries add 25-70 KB on top. For these four, protocol choice has minimal impact on flash.

2. **LwM2M is significantly larger** (1,155 KB — 26% more than IOTMP, 27% more than MQTT). The Anjay LwM2M stack includes CoAP, DTLS handshake logic, the full LwM2M object model engine, registration lifecycle, TLV encoding, and IPSO object handlers. This is the cost of a full device management protocol.

3. **MQTT has the smallest binary** (908 KB). The `esp_mqtt` library in ESP-IDF is highly optimized and minimal.

4. **IOTMP is in the middle** (939 KB) despite including significantly more protocol-level features: PSON encoder/decoder, resource model with I/O types, DESCRIBE handler, stream management, and compact mode support.

5. **Application code is 2.3-2.9× smaller with IOTMP.** All other programs require 109-139 lines for JSON/TLV serialization, command parsing, object/topic/URL routing, periodic publishing, and reconnection or registration handling. IOTMP requires 48 lines — resource definitions with lambdas.

6. **The other programs have less functionality.** Despite 2-3× more code, none includes: API discovery (DESCRIBE), server-controlled streaming (start/stop/interval), schema introspection, compact streaming mode, or bidirectional data streams.

7. **HTTP requires polling** for commands. CoAP relies on the device being reachable (NAT issues). LwM2M inherits CoAP's NAT limitations when using UDP.

### 2.2 Notes

- All five ESP-IDF programs use the same WiFi initialization code (`common_wifi.h`) to ensure identical baseline.
- No TLS is enabled in any program. Adding TLS would increase all binaries by ~100-150 KB equally.
- LwM2M required patching one mbedTLS API incompatibility (`mbedtls_ssl_conf_handshake_timeout` removed in mbedTLS 3.x). This is a known issue with the Anjay library version used.
- Source code for all programs is in `benchmarks/espidf/{iotmp,mqtt,coap,http,lwm2m}/`.
