## 2. Embedded Compilation Comparison (ESP32)

### 2.1 ESP-IDF (Primary — Same Framework Comparison)

All programs compiled with **ESP-IDF v5.4** for ESP32 using Docker (`espressif/idf:v5.4`).
Same framework, same toolchain, same target. Equivalent functionality:
- Connect to WiFi and authenticate with server/broker
- Define LED actuator (input resource / command handler)
- Define environment sensor (output: temperature + humidity)
- Define uptime resource
- Define reboot action
- Handle communication in main loop

#### Results

| Protocol | Image Size | Lines of Code | Library |
|---|---:|---:|---|
| **IOTMP** | **867,040 B** | **48** | iotmp-espidf + iotmp-embedded |
| MQTT | 907,776 B | 122 | esp_mqtt (built-in) + cJSON |
| HTTP | 909,348 B | 109 | esp_http_client (built-in) + cJSON |
| CoAP | 952,896 B | 128 | libcoap (espressif/coap component) |

#### Component Breakdown

| Component | IOTMP | MQTT | CoAP | HTTP |
|---|---:|---:|---:|---:|
| Protocol library | 13 KB | 14 KB | 83 KB | 24 KB |
| TLS/Crypto (mbedTLS) | ~30 KB | ~30 KB | ~30 KB | ~30 KB |

All programs include TLS support (mbedTLS) as part of the ESP-IDF component dependency chain. The protocol library row shows only the protocol-specific code; the bulk of every image (~830+ KB) is the shared ESP-IDF base (WiFi, TCP/IP, FreeRTOS, mbedTLS, newlib, etc.).

#### Key Observations

1. **IOTMP produces the smallest binary** (867 KB), 4.5% smaller than MQTT and 9% smaller than CoAP. The IOTMP libraries (iotmp-espidf + iotmp-embedded) contribute only ~13 KB of compiled protocol code, which includes PSON encoder/decoder, the full resource model, DESCRIBE handler, stream management, and compact mode support.

2. **All four protocols are within 10% of each other.** The ESP-IDF base (WiFi, TCP/IP, TLS, FreeRTOS) dominates at ~830+ KB. Protocol libraries add 13-83 KB on top. For practical purposes, protocol choice has modest impact on flash usage.

3. **CoAP has the largest protocol library** (83 KB). The libcoap component is a full-featured RFC 7252 implementation with block-wise transfer, observe, and DTLS support, which accounts for the larger contribution.

4. **Application code is 2.3-2.7x smaller with IOTMP.** All other programs require 109-128 lines for JSON serialization, command parsing, topic/URL routing, periodic publishing, and reconnection handling. IOTMP requires 48 lines -- resource definitions with lambdas.

5. **The other programs have less functionality.** Despite 2-3x more code, none includes: API discovery (DESCRIBE), server-controlled streaming (start/stop/interval), schema introspection, compact streaming mode, or bidirectional data streams.

6. **HTTP requires polling** for commands. CoAP relies on the device being reachable (NAT issues).

### 2.2 Notes

- All four ESP-IDF programs use the same WiFi initialization code (`common_wifi.h`) to ensure identical baseline.
- All programs include TLS support (mbedTLS) as part of the ESP-IDF component dependency chain.
- Source code for all programs is in `protocol/{iotmp,mqtt,coap,http}/`.
- Reproducible via `protocol/build.sh` (requires Docker).

## 3. Encoding Comparison (ESP32)

Payload: `{"temperature": 23.5, "humidity": 60, "pressure": 1013, "label": "outdoor"}`

All programs compiled with **ESP-IDF v5.4** for ESP32 using Docker (`espressif/idf:v5.4`).

#### Runtime Results

| Encoding | Image Size | Encoding Code | Wire Size | Encode | Decode | Implementation |
|---|---:|---:|---:|---:|---:|---:|
| PSON | 195,044 B | 2,589 B | 55 B | 10.00 us | 5.93 us | 344 lines |
| NanoCBOR | 196,076 B | 3,816 B | 53 B | 19.65 us | 16.86 us | 2,223 lines |
| TinyCBOR | 200,888 B | 6,604 B | 55 B | 20.04 us | 52.78 us | 5,619 lines |

PSON encodes 2x faster than CBOR and decodes 3-9x faster, with a standalone implementation that is 6-16x smaller.

#### Implementation Size

| Implementation | Core encoder+decoder | Total library |
|---|---:|---:|
| PSON (standalone C) | 344 | 344 |
| NanoCBOR | 1,124 | 2,223 |
| TinyCBOR | 2,354 | 5,619 |

PSON's 344 lines include the complete encoder and decoder as two standalone C headers (`pson_encode.h` + `pson_decode.h`). NanoCBOR and TinyCBOR totals include auxiliary utilities (string helpers, pretty-printing, etc.) beyond the core encoder/decoder.
